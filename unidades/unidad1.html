Fundamentos de Frontend

a Concepto de frameworks de Frontend web

Un framework de frontend es un conjunto de herramientas, librerías y convenciones que proporcionan una estructura base para desarrollar interfaces de usuario web. Estos frameworks permiten construir aplicaciones web complejas de manera más eficiente, organizada y mantenible.

Características principales:

  • Componentes reutilizables
  • Gestión de estado de la aplicación
  • Enrutamiento y navegación
  • Comunicación con APIs

Beneficios:

  • Desarrollo más rápido
  • Mayor seguridad
  • Trabajo en equipo facilitado
  • Responsive design integrado

PostCSS

Procesador CSS

Transforma CSS con JavaScript plugins. Tailwind lo usa internamente

postcss.config.js

Vite / Webpack

Bundlers

Empaquetadores de módulos para desarrollo y producción

npm run build

d Proceso de instalación y configuración de Tailwind CSS

Guía de instalación paso a paso:


# 1. Inicializar proyecto (si no existe)
npm init -y

# 2. Instalar Tailwind CSS
npm install -D tailwindcss postcss autoprefixer

# 3. Inicializar configuración
npx tailwindcss init -p

# 4. Configurar archivo tailwind.config.js
module.exports = {
  content: ["./src/**/*.{html,js,php}"],
  theme: {
    extend: {},
  },
  plugins: [],
}

# 5. Importar en CSS principal (styles.css)
@tailwind base;
@tailwind components;
@tailwind utilities;

# 6. Ejecutar en modo desarrollo
npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch

# 7. Compilar para producción
npx tailwindcss -i ./src/input.css -o ./dist/output.css --minify

Diseño Frontend

a Elementos que integran el framework de frontend

Los frameworks de frontend incluyen componentes, directivas, gestión de estado, y utilidades CSS para agilizar el desarrollo.

Componentes

Elementos UI reutilizables como botones, tarjetas, formularios

Directivas

Instrucciones especiales en el template (v-if, v-for)

Estado

Gestión de datos reactivos en la aplicación

b Modelo MVVM

MVVM (Model-View-ViewModel) es un patrón arquitectónico que separa la interfaz de usuario de la lógica de negocio. El ViewModel actúa como intermediario entre el Model (datos) y la View (interfaz).

c APIs y localStorage

Las APIs permiten la comunicación con servicios externos, mientras que localStorage proporciona almacenamiento persistente en el navegador del cliente.

d Hospedaje web

El hospedaje web implica servicios que almacenan y sirven sitios web en Internet, incluyendo hosting compartido, VPS, servidores dedicados y plataformas en la nube.

e Contenidos web

Los contenidos web incluyen texto, imágenes, videos, formularios y elementos interactivos que conforman la experiencia del usuario.

f UI/UX/IXD

UI (Interfaz de Usuario), UX (Experiencia de Usuario) e IXD (Diseño de Interacción) son disciplinas fundamentales para crear productos digitales centrados en el usuario.

g Diseño responsivo

El diseño responsivo permite que un sitio web se adapte automáticamente a diferentes tamaños de pantalla y dispositivos, mejorando la experiencia del usuario.
