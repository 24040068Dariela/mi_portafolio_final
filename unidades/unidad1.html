Fundamentos de Frontend

a Concepto de frameworks de Frontend web

Un framework de frontend es un conjunto de herramientas, librerías y convenciones que proporcionan una estructura base para desarrollar interfaces de usuario web. Estos frameworks permiten construir aplicaciones web complejas de manera más eficiente, organizada y mantenible.

Características principales:

  • Componentes reutilizables
  • Gestión de estado de la aplicación
  • Enrutamiento y navegación
  • Comunicación con APIs

Beneficios:

  • Desarrollo más rápido
  • Mayor seguridad
  • Trabajo en equipo facilitado
  • Responsive design integrado

d Proceso de instalación y configuración de Tailwind CSS

Guía de instalación paso a paso:


# 1. Inicializar proyecto (si no existe)
npm init -y

# 2. Instalar Tailwind CSS
npm install -D tailwindcss postcss autoprefixer

# 3. Inicializar configuración
npx tailwindcss init -p

# 4. Configurar archivo tailwind.config.js
module.exports = {
  content: ["./src/**/*.{html,js,php}"],
  theme: {
    extend: {},
  },
  plugins: [],
}

# 5. Importar en CSS principal (styles.css)
@tailwind base;
@tailwind components;
@tailwind utilities;

# 6. Ejecutar en modo desarrollo
npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch

# 7. Compilar para producción
npx tailwindcss -i ./src/input.css -o ./dist/output.css --minify

Configuración del archivo tailwind.config.js:

module.exports = { content: [ "./**/*.php", "./**/*.html", "./**/*.js", ], theme: { extend: { colors: { primary: '#6B3F69', secondary: '#8D5F8C', accent: '#A376A2', light: '#DDC3C3', dark: '#4A2D49', }, fontFamily: { 'poppins': ['Poppins', 'sans-serif'], }, }, }, plugins: [], }

Ejemplo de uso en HTML:

<!-- Botón con estilos de Tailwind --> <button class="bg-primary hover:bg-secondary text-white font-bold py-2 px-4 rounded-lg transition duration-300"> Click me </button> <!-- Tarjeta responsiva --> <div class="max-w-sm rounded-lg overflow-hidden shadow-lg bg-white"> <div class="px-6 py-4"> <h2 class="font-bold text-xl mb-2">Título</h2> <p class="text-gray-700">Contenido...</p> </div> </div>

Buenas prácticas de instalación:

  • Usar siempre la versión estable más reciente
  • Configurar PurgeCSS para eliminar CSS no utilizado en producción
  • Utilizar CDN solo para prototipos rápidos, no para producción
  • Mantener todas las dependencias actualizadas

b Modelo MVVM, instancias, interpolaciones, directivas, filtros, propiedades, funciones

Modelo MVVM (Model-View-ViewModel):

Model

Representa los datos y la lógica de negocio

View

Interfaz de usuario (HTML/CSS)

ViewModel

Conecta Model y View, maneja lógica de presentación

Elementos clave en Vue.js:

Instancias:

Cada aplicación Vue comienza con una instancia raíz

new Vue({ el: '#app' })
Interpolaciones:

Mostrar datos en templates con {{ }}

<p>{{ mensaje }}</p>
Directivas:

Atributos especiales con v-

<div v-if="visible"></div>

Ejemplo práctico:


<div id="app">
  <!-- Interpolación -->
  <h1>{{ titulo }}</h1>
  
  <!-- Directiva v-if -->
  <p v-if="mostrar">Este texto se muestra condicionalmente</p>
  
  <!-- Directiva v-for -->
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.nombre }}
    </li>
  </ul>
  
  <!-- Propiedades computadas -->
  <p>Total: {{ totalItems }}</p>
</div>

<script>
new Vue({
  el: '#app',
  data: {
    titulo: 'Mi Aplicación',
    mostrar: true,
    items: [
      { id: 1, nombre: 'Item 1' },
      { id: 2, nombre: 'Item 2' }
    ]
  },
  computed: {
    totalItems() {
      return this.items.length;
    }
  }
});
</script>

Ventajas del patrón MVVM:

  • Separación clara entre lógica y presentación
  • Fácil testing de componentes individuales
  • Reutilización de componentes ViewModel
  • Mantenimiento más sencillo del código

c Conceptos de localStorage, APIs y su consumo

localStorage vs sessionStorage:

Característica localStorage sessionStorage
Duración Persistente Por sesión
Alcance Mismo origen Misma pestaña
Límite 5-10MB 5-10MB
Uso en JavaScript:
// Guardar datos localStorage.setItem('usuario', JSON.stringify(usuario)); // Recuperar datos const usuario = JSON.parse(localStorage.getItem('usuario')); // Eliminar datos localStorage.removeItem('usuario'); // Limpiar todo localStorage.clear();

Consumo de APIs REST:

Las APIs permiten comunicación entre frontend y backend

Métodos HTTP comunes:
GET - Obtener datos POST - Crear datos PUT - Actualizar datos DELETE - Eliminar datos
Ejemplo con Fetch API:
// GET request fetch('https://api.ejemplo.com/usuarios') .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error)); // POST request fetch('https://api.ejemplo.com/usuarios', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ nombre: 'Juan', email: 'juan@ejemplo.com' }) });

Buenas prácticas:

Para localStorage:
  • No almacenar datos sensibles
  • Validar datos antes de guardar
  • Manejar cuotas de almacenamiento
Para APIs:
  • Manejar errores adecuadamente
  • Implementar loading states
  • Usar HTTPS en producción

f Definir diseños de Interfaces de Usuario (UI), Experiencias de Usuario (UX) y Diseño de Interacción (IXD)

Diferencias Clave:

UI (User Interface)

Diseño visual de la interfaz. Cómo se ve y se siente.

Colores Tipografía Iconos
UX (User Experience)

Experiencia completa del usuario. Cómo funciona y se usa.

Usabilidad Accesibilidad Satisfacción
IXD (Interaction Design)

Diseño de interacciones específicas. Cómo responde el sistema.

Animaciones Feedback Micro-interacciones

Wireframes

Definición

Estructura básica sin diseño visual

Objetivo

Definir layout y jerarquía de contenido

Herramientas

Figma, Balsamiq, Adobe XD

Características:

  • • Enfoque en funcionalidad
  • • Sin colores ni imágenes
  • • Rápido de crear/modificar
  • • Ideal para feedback temprano

Mockups

Definición

Diseño visual estático con detalles

Objetivo

Presentar apariencia final del diseño

Herramientas

Figma, Sketch, Adobe Photoshop

Características:

  • • Incluye colores y tipografía
  • • Muestra imágenes reales
  • • Representa branding
  • • Detalles visuales completos

Prototipado

Definición

Versión interactiva del diseño

Objetivo

Testear flujos de usuario y usabilidad

Herramientas

Figma, InVision, Proto.io

Características:

  • • Interactividad simulada
  • • Navegación entre pantallas
  • • Testeo con usuarios reales
  • • Iteraciones rápidas

g Identificar el framework de diseño responsivo a utilizar

Tailwind CSS como Framework Responsivo

Características Principales

Framework CSS utility-first

Diseño mobile-first por defecto
Breakpoints predefinidos
Clases utilitarias responsivas
Altamente personalizable
Breakpoints por Defecto

Puntos de ruptura estándar

sm ≥640px (Tablets pequeñas) Mobile
md ≥768px (Tablets) Tablet
lg ≥1024px (Laptops) Desktop
xl ≥1280px (Pantallas grandes) Large

Implementación Responsiva en HTML

Grid System Responsivo:
<!-- Grid que cambia columnas según tamaño --> <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4"> <div class="bg-blue-100 p-4">Item 1</div> <div class="bg-blue-100 p-4">Item 2</div> <div class="bg-blue-100 p-4">Item 3</div> </div>

• 1 columna en móvil
• 2 columnas en tablet
• 3 columnas en desktop

Mostrar/Ocultar Elementos:
<!-- Visible solo en desktop --> <div class="hidden md:block"> Contenido solo visible en tablets y mayores </div> <!-- Visible solo en móvil --> <div class="md:hidden"> Contenido solo visible en móvil </div>

Tipografía y Espaciado Responsivo

Tamaños de Texto Adaptativos:
<!-- Texto que crece con el viewport --> <h1 class="text-2xl md:text-3xl lg:text-4xl xl:text-5xl"> Título Responsivo </h1> <p class="text-sm md:text-base lg:text-lg"> Párrafo con tamaño adaptable </p>
Padding y Margin Responsivos:
<!-- Espaciado que varía por breakpoint --> <div class="p-4 md:p-6 lg:p-8"> <!-- Menos padding en móvil, más en desktop --> Contenido con espaciado adaptable </div> <div class="mt-4 md:mt-8 lg:mt-12"> <!-- Margen superior creciente --> Sección con márgenes responsivos </div>

Buenas Prácticas con Tailwind CSS

Mobile-First Approach:
  • Diseñar primero para móvil
  • Usar breakpoints para mejoras progresivas
  • Evitar hide/show innecesario
Performance:
  • Purge CSS para producción
  • Minificar archivos finales
  • Usar CDN para desarrollo rápido
Configuración Personalizada:
// tailwind.config.js module.exports = { theme: { screens: { 'sm': '640px', 'md': '768px', 'lg': '1024px', 'xl': '1280px', '2xl': '1536px', }, extend: { // Breakpoints personalizados si es necesario } } }

Ventajas de Tailwind CSS para Responsive Design

  • Desarrollo más rápido con clases utilitarias
  • Código más limpio y mantenible
  • Consistencia en todos los breakpoints
  • Fácil personalización del diseño system
