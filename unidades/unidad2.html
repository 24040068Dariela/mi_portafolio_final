1. Fundamentos de Backend

a Concepto de frameworks de backend web

Un framework de backend es una plataforma de software que proporciona una estructura base para desarrollar la lógica del servidor de aplicaciones web. Estos frameworks facilitan tareas como el enrutamiento, la gestión de bases de datos, la autenticación de usuarios y la creación de APIs.

Funciones principales:

  • Gestión de peticiones HTTP
  • Conexión con bases de datos
  • Autenticación y autorización
  • Manejo de sesiones

Ejemplos populares:

  • Express.js (Node.js)
  • Django (Python)
  • Laravel (PHP)
  • Ruby on Rails (Ruby)

b Tipos de frameworks de backend existentes

Framework Lenguaje Tipo Características
Express.js JavaScript Minimalista Ligero, flexible, gran ecosistema
Django Python Completo (baterías incluidas) ORM, admin automático, seguro
Laravel PHP Completo Eloquent ORM, Blade templating
Spring Boot Java Empresarial Robusto, escalable, seguro
Flask Python Microframework Ligero, flexible, extensible

d Proceso de instalación y configuración de Express.js

Guía de instalación paso a paso:


# 1. Instalar Node.js (incluye npm)
# Descargar desde https://nodejs.org/

# 2. Verificar instalación
node --version
npm --version

# 3. Inicializar proyecto
mkdir mi-backend
cd mi-backend
npm init -y

# 4. Instalar Express
npm install express

# 5. Instalar dependencias de desarrollo (opcional)
npm install -D nodemon

# 6. Crear archivo principal (app.js o index.js)
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('¡Hola Mundo desde Express!');
});

app.listen(port, () => {
  console.log(`Servidor corriendo en http://localhost:${port}`);
});

# 7. Ejecutar servidor
node app.js
# O con nodemon para desarrollo
npx nodemon app.js

2. Diseño Backend

a Elementos que integran el framework de backend

i ¿Qué es MVC?

MVC (Model-View-Controller) es un patrón arquitectónico que separa una aplicación en tres componentes principales:

  • Modelo (Model): Gestiona los datos y la lógica de negocio. Interactúa con la base de datos.
  • Vista (View): Presenta los datos al usuario (interfaz). En backend, pueden ser plantillas HTML o respuestas JSON.
  • Controlador (Controller): Maneja las peticiones del usuario, interactúa con el modelo y selecciona la vista adecuada.

ii Migraciones

Las migraciones son una forma controlada y versionada de modificar la estructura de la base de datos. Permiten crear, modificar o eliminar tablas y columnas de manera programática.

Ejemplo con Sequelize (ORM para Node.js):

npx sequelize-cli model:generate --name User --attributes name:string,email:string npx sequelize-cli db:migrate

iii Modelos de Bases de Datos

Los modelos son representaciones de las tablas de la base de datos en el código. Definen la estructura, relaciones y comportamientos de los datos.

Ejemplo con Mongoose (MongoDB):

const userSchema = new mongoose.Schema({ name: String, email: { type: String, unique: true }, age: Number });

Ejemplo con Sequelize (SQL):

const User = sequelize.define('User', { name: DataTypes.STRING, email: DataTypes.STRING });

iv Vistas y rutas

Las rutas definen los endpoints de la API y asocian URLs con controladores. Las vistas en backend pueden ser plantillas HTML (SSR) o respuestas JSON (API).

// Definición de rutas en Express app.get('/api/users', userController.getAllUsers); app.post('/api/users', userController.createUser); app.get('/api/users/:id', userController.getUserById);

v Controladores

Los controladores contienen la lógica de negocio de la aplicación. Procesan las peticiones, interactúan con los modelos y retornan respuestas.

// Ejemplo de controlador en Express const userController = { getAllUsers: async (req, res) => { try { const users = await User.find(); res.json(users); } catch (error) { res.status(500).json({ error: error.message }); } } };

b UI/UX/IXD en el desarrollo backend

Aunque UI/UX/IXD se asocian principalmente con el frontend, en backend también son importantes:

UI (Interfaz)

Diseño de APIs claras y consistentes, estructura de endpoints RESTful, formatos de respuesta predecibles.

UX (Experiencia)

Tiempos de respuesta optimizados, manejo de errores claro, autenticación fluida, caché inteligente.

IXD (Interacción)

WebSockets para tiempo real, notificaciones push, flujos de datos bidireccionales.

3. Desarrollo de Backend

a Gestor de base de datos

Un gestor de base de datos (DBMS) es el software que permite crear, administrar y manipular bases de datos. Para nuestro proyecto con Express, podemos usar:

MongoDB (NoSQL)

Base de datos documental, ideal para datos no estructurados. Se usa con Mongoose.

MySQL / PostgreSQL (SQL)

Bases de datos relacionales, ideales para datos estructurados. Se usan con Sequelize.

b Modelos conceptuales de base de datos

Un modelo conceptual representa las entidades, atributos y relaciones del sistema, independientemente de la tecnología.

Ejemplo: Modelo E-R para un blog

Usuario

  • • id (PK)
  • • nombre
  • • email
  • • password

Publicación

  • • id (PK)
  • • título
  • • contenido
  • • usuario_id (FK)

Comentario

  • • id (PK)
  • • texto
  • • publicación_id (FK)
  • • usuario_id (FK)

Relaciones: Usuario (1) ── (N) Publicación (1) ── (N) Comentario

c Diseños de interfaces de usuario

Aunque el backend no genera directamente la interfaz, define qué datos estarán disponibles para el frontend. El diseño de APIs RESTful debe considerar:

  • Endpoints intuitivos: /api/users, /api/posts, /api/posts/:id/comments
  • Métodos HTTP apropiados: GET, POST, PUT, DELETE
  • Estructura de respuesta consistente: { success: true, data: [...], error: null }
  • Códigos de estado HTTP: 200, 201, 400, 401, 404, 500

d Funcionalidad del sitio con el framework

Con Express.js podemos implementar todas las funcionalidades del backend:

CRUD completo

Crear, leer, actualizar y eliminar recursos mediante APIs REST.

Autenticación

Registro, login, JWT, sesiones, roles y permisos.

Middleware

Validación, logging, manejo de errores, CORS, compresión.

WebSockets

Comunicación en tiempo real con Socket.io.
